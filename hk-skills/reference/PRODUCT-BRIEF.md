# PRODUCT-BRIEF — AI Outbound 금융상품 Sales Call Bot

> **Single source of truth for what we are building.**
> **우리가 만드는 것에 대한 단일 진실 공급원입니다.**

---

## 1. 한 문장 요약 / One-line Summary

콜센터 관리자가 대시보드에서 금융상품 아웃바운드 대상 고객의 대기열과 콜 진행 상태를 모니터링하고, AI콜봇이 고객에게 자동 발신하여 상품을 1차 안내한 뒤, 고객이 한도조회 등
상담원 응대가 필요한 요청을 하면 상담원에게 통화를 이관하는 해커톤 데모 시스템

---

## 2. 사용자 / Users

- ** 주 사용자 : 콜센터 관리자 - 한국어
- ** 시스템 행위자 : AI콜봇 - 한국어 음성 응대
- ** 대상 : 금융 상품 아웃바운드 TM 대상 고객 - 데모에서는 가짜 iPhone UI로 표현, 한국어
- ** 시청자 : 해커톤 심사위원 - 데모 평가자

---

## 3. 핵심 시나리오 / Core Scenarios

통화 시나리오는 총 2개로 구성합니다. 

| ID | 이름 | 트리거 조건 | 상담원 행동 |
|---|---|---|---|
| S1 | **상품관심, 한도조회 요청** | 고객이 대환대출, 자동차담보대출 조건 비교에 관심을 보이거나, 한도조회/상담원 연결을 요청 | 상담원 연결 상태로 전환 |
| S2 | **보이스피싱 피해 의심** | LLM이 송금 요구, 앱 설치 요구, 개인정보·인증번호 요청 등 금융사기 패턴 감지 | AI콜봇이 피싱 위험을 안내하고 통화 종료 |

S1은 상담원 연결 상태로 전환됩니다.
S2는 상담원 연결 없이 AI콜봇이 피싱 위험을 안내한 뒤 통화 종료됩니다.

## 4. 화면 구성 / Screens

### 4.1 상담원 대시보드 (Agent UI)

- **Outbound Call Queue 테이블**
  - 색상 변화: 노란(전화중) / 검정(무응답) / 갈색(거절) / 초록(가입희망) / 빨강(상담원연결요청/Fraud)
  - 빨강/초록은 **요청 시점부터 경과 시간 내림차순으로 고정**
- **통화 화면** (이전 화면에서 행 클릭 시 진입)
  - **좌측 수직**: 통화 노드 그래프 (React Flow). 클릭 시 해당 노드의 transcript가 펼쳐짐.
  - **우측 상단**: LLM 가이드라인 + 통화 연결 사유
  - **우측 하단**: 고객 페르소나/신용/금융 카드
  - **최하단**: 상품 드롭다운 + **가입 승인** 버튼 (거부 버튼은 없음 — 통화 종료는 고객이 결정)
- **통화 종료 후 팝업**
  - LLM이 작성한 메모 초안 (수정 가능)
  - 상담 결과 유형 라디오 박스
  - 확인 시 DB 저장 + 첫 화면 복귀
 
- **관리자 대시보드**
  - 요약 카드 : 대기콜 수, 진행 중 통화 수, 상담원 연결 필요 수, 금융사기 의심 수, 종료된 통화 수
  - 아웃바운드 대기콜 리스트 : 
    - 1. AI콜봇이 자동 발신 예정인 대기콜 리스트 표시 
    - 2. 관리자가 행을 클릭하면 실시간 통화 모니터링 화면으로 진입
    - 3. 클릭은 콜 시작이 아니라 모니터링 화면 진입 동작
    - 4. 기본 상태는 일반 리스트로 표시
    - 5. 상담원 연결 필요 또는 금융사기 의심처럼 확인이 필요한 건만 강조 표시
    - 6. 강조된 항목은 발생 시점 기준으로 상단에 노출

- **실시간 통화 모니터링 화면**
  - (좌측) 고객 휴대폰 화면
  - (우측) AI 분석 패널
    - 1. 고객 기본 정보 : 고객명, 대상상품, 금리/한도, 기존 대출 정보(당사/타사), 차량 보유 여부
    - 2. 실시간 분석 : 이탈 위험도(강조표시), 감정상태, DATA(실시간 정보+기존 정보)
    - 3. 판단 근거 : 고객 발화와 기존 정보를 바탕으로 AI가 판단한 근거 표시
    - 4. AI 추천 : 추천 코멘트, 추천 Action(강조표시) 
    - 5. 분기 처리 : 고객이 한도조회/상담원 연결을 요청하면 상담원 연결 상태로 전환 / 금융사기 의심 발화가 감지되면 AI콜봇이 위험을 안내하고 통화 종료

- **통화 종료 및 인계 요약 화면**
  - (좌측) 대화 이력 : AI콜봇과 고객의 전체 대화 내용 표시(중요 부분 강조 표시) 
  - (우측) 통화 요약 : 
    - 1. AI가 고객 반응, 주요 발화, 최종 의사를 종합해 상담 요약을 표시
    - 2. 상담원 연결 사유 또는 금융사기 위험안내 완료 여부를 함께 표시

### 4.2 고객 iPhone UI (Customer Phone UI)

- 1. 스타일 : 아이폰 스타일의 전화 수신 화면을 별도 창 또는 탭으로 표시
- 2. 수신 화면 : 받기 / 거절 버튼 제공
- 3. 통화 화면 : 받기 선택 시 통화 화면으로 전환되며, 통화 시간 타이머와 종료 버튼 표시
- 4. 사운드 재생 : 수신음과 통화 음성은 실제 휴대폰이 아니라 노트북 스피커로 재생
- 5. 실제 고객 휴대폰에 전화가 걸리는 것은 아니며, 데모용 화면으로 통화 상황을 시뮬레이션

### 4.3 마이크 채널 토글 (Mic Channel Toggle)

- 1. 고객은 노트북에 연결된 마이크로 발화
- 2. 고객 음성은 STT를 통해 텍스트로 변환되어 AI콜봇이 인식
- 3. AI콜봇 응답은 TTS로 변환되어 노트북 스피커로 재생
- 4. 대화 기록에는 고객 발화는 STT 결과로, AI콜봇 발화는 TTS 출력문으로 자동 구분되어 표시
- 5. 별도 화자 토글 없이 AI콜봇 / 고객 화자 라벨을 자동 반영

## 5. 비기능 요구 사항 / Non-Functional

| 항목 | 요구 | 이유 |
|---|---|---|
| 데모 환경 | 로컬 노트북 (단일 머신) | 24h 안에 클라우드 배포는 risk 큼 |
| 다국어 | 한국어 primary | 심사위원/사용자 한국어 |
| LLM latency | 첫 토큰 < 2초 | 실시간 통화 UX |
| STT latency | < 2초 (chunk 2-3s) | 발화 후 자연스러운 응답 |
| TTS latency | < 1.5초 | 끊김 없는 통화 |
| 데이터 | SQLite (단일 파일) | 설치 단순, 백업 = 파일 복사 |
| 동시 사용자 | 1 상담원 + 1 고객 (데모) | 24h 안에 멀티 유저는 over |

| 항목 | 요구사항 | 이유 |
|---|---|---|
| 실행 환경 | 	로컬 노트북 단일 실행 | 해커톤 기간 내 빠른 세팅과 안정적인 데모 실행을 위함
| 배포 방식 | 	클라우드 배포 없이 로컬 실행 기준	| 24시간 내 클라우드 배포까지 진행하기에는 리스크가 큼
| 언어 | 	한국어 우선 지원 | 관리자 화면, 고객 발화, AI콜봇 응답 모두 한국어 기준
| LLM 응답 지연 | 	첫 토큰 2초 이내 목표 | 실시간 통화에서 응답 대기 시간이 길어지지 않도록 하기 위함
| STT 지연 | 	고객 발화 후 2초 이내 텍스트 변환 목표	| 고객 발화 직후 AI콜봇이 자연스럽게 응답하기 위함
| TTS 지연 | 	응답 생성 후 1.5초 이내 음성 재생 목표	| 통화 흐름이 끊기지 않도록 하기 위함
| 데이터 저장 | 	DuckDB 단일 파일 사용	| 로컬 실행에 적합하고, 통화 로그·분석 결과·요약 데이터를 파일 기반으로 관리하기 쉬움
| 동시 사용자 | 	관리자 1명 + 고객  1명 기준 | AI콜봇은 사용자가 아니라 시스템 내부 행위자로 간주하며, 멀티 유저 기능은 이번 데모 범위에서 제외
| 장애 대응 | 	API 실패 시 기본 안내 문구 또는 예비 시나리오 사용 | 음성/STT/LLM/TTS 오류가 발생해도 데모 흐름을 유지하기 위함

## 6. Out of Scope (24h 안에 안 함)

- ❌ 실제 전화망 연동 — Twilio, 통신사 전화망, 실제 휴대폰 발신은 구현하지 않음
- ❌ 실제 휴대폰 착신 — 고객 화면은 별도 브라우저 창 또는 탭의 가짜 iPhone 화면으로 시뮬레이션
- ❌ 다중 사용자 동시 사용 — 관리자 1명 + 고객 1명 기준으로 진행
- ❌ 상담원 전용 인입 큐 — 상담원 연결 상태 전환까지만 표현
- ❌ 실제 상담원 통화 연결 — 실제 연결 없이 인계 상태만 표시
- ❌ 인증/권한 — 로그인, 계정, 역할별 권한은 구현하지 않음
- ❌ 모바일 반응형 — 데스크톱/노트북 화면 기준으로 구현
- ❌ 영문 발화 — 고객 발화와 AI콜봇 응답은 한국어만 지원
- ❌ 화자 분리 모델 — 고객 STT와 AI콜봇 TTS 출력문으로 구분
- ❌ 푸시 알림, 이메일, SMS — 외부 알림 채널 연동은 구현하지 않음
- ❌ 운영 모니터링/로깅 시스템 — 콘솔 로그 수준으로 처리
- ❌ LLM 파인튜닝 — 프롬프트 기반으로 구현하고 별도 모델 학습은 진행하지 않음
- ❌ 프롬프트 A/B 테스트 — 데모용 단일 프롬프트 흐름으로 고정
- ❌ 외부 금융/신용정보 API 연동 — 고객 정보와 상품 조건은 가상 데이터 기준으로 사용

## 7. 성공 기준 / Success Criteria

- 시나리오 #1 : 
  - 상품 관심 / 한도조회 요청
  - 1. 고객이 조건 비교, 한도조회 또는 상담원 연결 의사를 표현
  - 2. LLM이 상품 관심 또는 한도조회 요청을 감지
  - 3. AI콜봇이 상담원 연결 상태로 전환
  - 4. 통화 종료 후 대화 이력과 요약 표시
   
- 시나리오 #2 : 
  - 보이스피싱 피해 의심
  - 1. 고객이 송금 요구, 앱 설치 요구, 개인정보·인증번호 요청 등 금융사기 의심 발화를 함
  - 2. LLM이 금융사기 위험 패턴을 감지
  - 3. AI콜봇이 피싱 위험을 안내하고 통화 종료
  - 4. 통화 종료 후 대화 이력과 요약 표시

- 공통 성공 조건 : 
  - 고객 음성 입력 → STT → LLM 판단 → AI콜봇 응답 → TTS 출력 흐름이 동작
  - 시나리오별 최종 상태가 정상 반영됨
  - 관리자 대시보드, 고객 휴대폰 시뮬레이션, 실시간 통화 모니터링 화면, 통화 종료 및 인계 요약 화면이 자연스럽게 이어짐
