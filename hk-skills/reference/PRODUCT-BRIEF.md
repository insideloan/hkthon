# PRODUCT-BRIEF — AI Outbound 금융상품 Sales Call Bot

> **Single source of truth for what we are building.**
> **우리가 만드는 것에 대한 단일 진실 공급원입니다.**

---

## 1. 한 문장 요약 / One-line Summary

콜센터 관리자가 대시보드에서 금융상품 아웃바운드 대상 고객의 대기열과 콜 진행 상태를 모니터링하고, AI콜봇이 고객에게 자동 발신하여 상품을 1차 안내한 뒤, 고객이 한도조회 등
상담원 응대가 필요한 요청을 하면 상담원에게 통화를 이관하는 해커톤 데모 시스템. 모든 상담이 끝나면 상담 내용 summary가 생성되며, 그 안에는 상담 flow 요약, 카테고라이징, STT contents가 구성된다. summary 화면은 상담원이 통화를 인계받을 때 참고하며, 추후 기술과 상담이 잘 작동했는지 모니터링할 때 사용된다.

## 4. 화면 구성 / Screens

### 4.1 상담원 대시보드 (Agent UI)
 
- **관리자 대시보드**
  - (상단) 요약 카드 : 대기콜 수, 진행 중 통화 수, 상담원 연결 필요 수, 금융사기 의심 수, 종료된 통화 수
  - (하단) 아웃바운드 대기콜 리스트 : 
    - AI콜봇이 자동 발신 예정인 대기콜 리스트 표시 
    - 관리자가 행을 클릭하면 실시간 통화 모니터링 화면으로 진입
    - 클릭은 콜 시작이 아니라 모니터링 화면 진입 동작
    - 기본 상태는 일반 리스트로 표시
    - 상담원 연결 필요 또는 금융사기 의심처럼 확인이 필요한 건만 강조 표시 (빨강색) 
    - 강조된 항목은 해당 상태로 전환된 시점을 기준으로 경과 시간 내림차순 정렬해 상단에 고정

- **실시간 통화 모니터링 화면 | 이전 화면에서 행 클릭 시 진입**
  - (좌측) 고객 휴대폰 화면
  - (우측) AI 분석 패널
    - (헤드) 시스템 명칭, 통화 시간 표시
    - (좌측 상단) 고객 정보  : 고객명, 대상상품, 금리/한도, 기존 대출 정보(당사/타사), 차량 보유 여부
    - (우측 상단) INDEX : 이탈 위험도 숫자, 게이지 강조 표시, 고객 감정
    - (중앙 메인) AI Action : 추천 Comment 및 Action 강조 표시
    - (하단) 판단 근거 : AI가 판단한 근거 요약 표시, DATA (실시간 정보+기존 정보)
    - (최하단) 문자URL발송 또는 상담원연결 버튼 : 고객 요청 사항에 따라 AI콜봇이 버튼 작동
    - 분기 처리 : 고객이 한도조회/상담원 연결을 요청하면 상담원 연결 상태로 전환 / 금융사기 의심 발화가 감지되면 AI콜봇이 위험을 안내하고 통화 종료
  - AI 분석 패널 레이아웃 (참고) :
    ```
    [좌상: 고객정보]            [우상: 이탈위험도/감정 - 강조]
    [중앙 메인: AI Action - 가장 크게]
    [하단: 판단 근거 | 실시간 DATA]
    ```

- **통화 종료 및 인계 요약 화면 | 이전 화면에서 최하단 버튼 클릭 시 진입**
  - (좌측) 대화 이력 : AI콜봇과 고객의 전체 대화 내용 표시(중요 부분 강조 표시) 
  - (우측) 통화 요약 : 
    - AI가 고객 반응, 주요 발화, 최종 의사를 종합해 상담 요약을 표시
    - 상담원 연결 사유 또는 금융사기 위험안내 완료 여부를 함께 표시

### 4.2 고객 iPhone UI (Customer Phone UI)

- 스타일 : 아이폰 스타일의 전화 수신 화면을 별도 창 또는 탭으로 표시
- 수신 화면 : 받기 / 거절 버튼 제공
- 통화 화면 : 받기 선택 시 통화 화면으로 전환되며, 통화 시간 타이머와 종료 버튼 표시
- 사운드 재생 : 수신음과 통화 음성은 실제 휴대폰이 아니라 노트북 스피커로 재생
- 실제 고객 휴대폰에 전화가 걸리는 것은 아니며, 데모용 화면으로 통화 상황을 시뮬레이션

### 4.3 마이크 채널 토글 (Mic Channel Toggle)

- 고객은 노트북에 연결된 마이크로 발화
- 고객 음성은 STT를 통해 텍스트로 변환되어 AI콜봇이 인식
- AI콜봇 응답은 TTS로 변환되어 노트북 스피커로 재생
- 대화 기록에는 고객 발화는 STT 결과로, AI콜봇 발화는 TTS 출력문으로 자동 구분되어 표시
- 별도 화자 토글 없이 AI콜봇 / 고객 화자 라벨을 자동 반영

## 5. 비기능 요구 사항 / Non-Functional

| 항목 | 요구사항 | 이유 |
|---|---|---|
| 실행 환경 | 로컬 노트북 (단일 실행) | 빠른 세팅과 안정적 데모 실행을 위함
| 배포 방식 | 클라우드 배포 없이 로컬 실행 기준	| 24시간 내 클라우드 배포는 리스크 큼
| 언어 | 한국어 우선 지원 | 사용자/심사위원 한국어
| LLM 응답 지연 | 첫 토큰 < 2초 | 실시간 통화 UX
| STT 지연 | 고객 발화 후 < 2초 (chunk 2-3s)	| 발화 후 자연스러운 응답
| TTS 지연 | 응답 생성 후 < 1.5초 | 끊김 없는 통화
| 데이터 저장 | DuckDB (단일 파일)	| 별도 DB 서버 없이 실행 가능, 파일 저장 및 조회 용이
| 동시 사용자 | 1 관리자 + 1 고객 (데모) | AI콜봇은 시스템 내부 행위자로 간주, 멀티 유저 기능은 데모 범위에서 제외
| 장애 대응 | API 실패 시 기본 안내 문구 | 음성/STT/LLM/TTS 오류 발생 시 확인을 위함

## 6. Out of Scope (24h 안에 안 함)

- ❌ 실제 전화망 연동 — Twilio, 통신사 전화망, 실제 휴대폰 발신은 구현하지 않음
- ❌ 실제 휴대폰 착신 — 고객 화면은 별도 브라우저 창 또는 탭의 가짜 iPhone 화면으로 시뮬레이션
- ❌ 다중 사용자 동시 사용 — 관리자 1명 + 고객 1명 기준으로 진행
- ❌ 상담원 전용 인입 큐 — 상담원 연결 상태 전환까지만 표현
- ❌ 실제 상담원 통화 연결 — 실제 연결 없이 인계 상태만 표시
- ❌ 인증/권한 — 로그인, 계정, 역할별 권한은 구현하지 않음
- ❌ 모바일 반응형 — 데스크톱/노트북 화면 기준으로 구현
- ❌ 영문 발화 — 고객 발화와 AI콜봇 응답은 한국어만 지원
- ❌ 화자 분리 모델 — 고객 STT와 AI콜봇 TTS 출력문으로 구분
- ❌ 푸시 알림, 이메일, SMS — 외부 알림 채널 연동은 구현하지 않음
- ❌ 운영 모니터링/로깅 시스템 — 콘솔 로그 수준으로 처리
- ❌ LLM 파인튜닝 — 프롬프트 기반으로 구현하고 별도 모델 학습은 진행하지 않음
- ❌ 프롬프트 A/B 테스트 — 데모용 단일 프롬프트 흐름으로 고정
- ❌ 외부 금융/신용정보 API 연동 — 고객 정보와 상품 조건은 가상 데이터 기준으로 사용

## 7. 성공 기준 / Success Criteria

- 시나리오 #1 : 
  - 상품 관심 / 한도조회 요청
  - 고객이 조건 비교, 한도조회 또는 상담원 연결 의사를 표현
  - LLM이 상품 관심 또는 한도조회 요청을 감지
  - AI콜봇이 상담원 연결 상태로 전환
  - 통화 종료 후 대화 이력과 요약 표시

- 공통 성공 조건 : 
  - 고객 음성 입력 → STT → LLM 판단 → AI콜봇 응답 → TTS 출력 흐름이 동작
  - 시나리오별 최종 상태가 정상 반영됨
  - 관리자 대시보드, 고객 휴대폰 시뮬레이션, 실시간 통화 모니터링 화면, 통화 종료 및 인계 요약 화면이 자연스럽게 이어짐
