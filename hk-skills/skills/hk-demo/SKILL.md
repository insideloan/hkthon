# hk-demo — 데모 준비 / Demo Prep

> **목적 / Purpose**: 심사위원 앞에서 보여줄 4분 데모를 **실패 없이** 진행하기 위해 시나리오를 굳히고 리허설.
> Lock in the 4-minute demo flow and rehearse so it doesn't fail in front of judges.

---

## 1. 언제 쓰나 / When to use

- 모든 P0 slice가 done이고, 통합 데모가 **사용자 1명이 처음부터 끝까지 1번 성공** 했을 때.
- 해커톤 종료 2시간 전 (이상적으로).

**트리거**:
- "데모 준비하자" / "demo prep"
- "발표 순서 정하자"

---

## 2. 입력 / Input

- `PRODUCT-BRIEF.md` (성공 기준 §7)
- `BACKLOG.md` (Demo Plan §)
- 본인 환경: backend + frontend 둘 다 실행 가능
- (선택) 발표 시간 (예: 4분)

---

## 3. 진행 / Process

### 3.1 데모 시나리오 작성

`DEMO.md` 작성 (`~/workspace/hackathon-2026/DEMO.md`):

```markdown
# 데모 시나리오 (4분)

## 타임라인

| 시간 | 무엇 | 누가 | 화면 | 스크립트 |
|---|---|---|---|---|
| 0:00 | intro | 발표자A | 슬라이드 1 | "AI가 직접 전화를 걸고 세일즈하는 시스템입니다" |
| 0:15 | queue 화면 | 발표자A | agent UI | "관리자가 고객 목록을 봅니다" |
| 0:30 | 클릭 | 발표자A | customer iPhone | "이 분께 전화를 겁니다" → iPhone이 울림 |
| 0:45 | S1 happy path | (자동) | 양쪽 | AI 봇이 인사 → 상품 소개 → 가입 권유 → 고객 OK |
| 1:30 | 상담원 인계 | 발표자B | agent UI | 빨간/초록 행 → 클릭 → 통화 화면 |
| 1:50 | 노드 그래프 | 발표자B | agent UI 통화 | "AI는 이 흐름으로 통화를 진행했고, 여기서 상담원 인계 요청이..." |
| 2:10 | 페르소나 + 승인 | 발표자B | agent UI 통화 | "고객의 신용도는 700이고, 이 상품 승인" |
| 2:30 | 통화 종료 + 인계 요약 | 발표자A | agent UI 통화 | "고객이 끊으면 AI 인계 요약 화면, LLM이 통화 내용 자동 요약" |
| 3:10 | 마무리 | 발표자A | 슬라이드 | "감사합니다" |
```

### 3.2 폴리시 체크리스트 (5개 영역)

DEMO.md 작성하면서 동시에 다음 5가지를 점검:

#### A. 데모 데이터 시드
- `python -m app.seed`로 **테스트용 고객 10명**이 들어가 있는가
- 그 중 최소 1명은 S1 시나리오에 맞춰 페르소나가 세팅되어 있는가
- (이상적으로) 시드 직후 첫 손님이 S1 happy path로 흘러가게

#### B. 폴백 시나리오
- 데모 중 뭔가 망가지면? 다음 3개 fallback:
  1. **LLM timeout** → 5초 안에 안 오면 미리 준비해둔 hardcoded script로 자동 전환 (state machine에 fallback node)
  2. **STT/TTS 안 됨** → 그냥 "음성" 부분 빼고 transcript + 그래프만 보여주기 (시각적 데모)
  3. **WebSocket 끊김** → 새로고침 자동 재연결 (frontend에 reconnect logic 있어야)

#### C. 브라우저/탭 셋업
- 발표 직전 chrome 창 2개:
  - 창 1: agent UI (`/`)
  - 창 2: customer iPhone UI (`/phone`)
- 발표자 둘이서 둘 다 만질 수 있게
- 혹시 둘이 못 만지면, 발표자 1명이 alt-tab으로 전환 (느리지만 가능)
- 알림 끄기 (Mac의 "Do Not Disturb")
- 화면 밝기 max

#### D. 폴리시 (있으면 좋음, 24h 시간 없으면 skip)
- **마이크** 1개로 양쪽 발화 — 토글 위치 큰 버튼으로
- **TTS 음성**이 너무 기계적이면 Typecast `model=ssfm-v30` 확인 또는 voice `혜라`/`진서`/`유라` 중 다른 화자로 교체
- **노드 그래프**에 LLM summary가 너무 짧으면 system prompt 수정
- **summary panel**의 LLM 요약이 너무 일반적 → 예시 1-2개 시드에 심어두기
- **상담원 인계 빨간/초록 행**이 잘 보일지 (대비, 폰트)
- **타이머** (통화 시간)가 잘 동작하는지

#### E. 시간 엄수
- **4분 안에 끝나야 함.** 4분 30초 넘으면 보통 감점.
- 리허설 1번: 0:00 시작해서 어디서 끊을지 (예: 요약 화면까지 보여주고 끝내기)
- 시간이 부족하면 노드 그래프 + 인계 요약 부분을 줄이는 것도 option

### 3.3 리허설 (15-30분)

**1회 통으로** 리허설:

1. **타이머 시작** → 0:00부터 4:00까지
2. **시나리오대로** 진행 (DEMO.md)
3. **실수/버그/시간 초과** 메모
4. **끝나고 5분 discussion**:
   - 뭐가 잘 됐나
   - 뭐가 막혔나
   - 어디를 더 polish할까 (시간 있으면)
5. **2회차 리허설** (시간 있으면)

### 3.4 최종 점검 (5분)

리허설 끝나고 발표 직전:

- [ ] `git status` — 의도하지 않은 변경 없는지
- [ ] DB가 시드 상태인지 (summary 등이 데모 시점에 비어있게)
- [ ] chrome 창 2개, 주소 정확히
- [ ] 발표자 A/B의 역할 명확
- [ ] fallback 시나리오 2개 팀원 모두 인지
- [ ] 4분 안에 끝난다는 자신감

## 5. 발표 중 팁 / Presentation Tips

1. **첫 15초가 가장 중요** — "이게 뭔가요?"가 한 문장으로. 심사위원 1명당 집중 시간 1분.
2. **기술 자랑 < 사용자 가치** — "저희는 React 19, FastAPI, WebSocket streaming, AWS Transcribe/Typecast를..." ❌ → "AI가 직접 전화를 걸고, 한도조회 요청 시 상담원에게 자동으로 연결합니다" ⭕
3. **데모가 1번 망가져도 멘탈 유지** — "아, 잘 보이시죠? (재시도)" 한 마디로 살림.
4. **숫자 3개 외우기** — 예: "0.8초 응답, 상담원 자동 연결, 4분 안에 풀 데모"
5. **질문 시간에 "잘 모르겠습니다" OK** — 없는 knowledge를 지어내는 것보다 100배 낫다.

## 6. 가드레일 / Guardrails

- ❌ **데모 중 live coding 금지** — 미리 다 짜두고 시연만.
- ❌ **새 기능 추가 시도 금지** — 24h 종료 직전 새 시도 = disaster.
- ❌ **fallback 없이 데모 시작 금지** — LLM timeout, STT 끊김, WebSocket 끊김 중 1개는 거의 100% 일어남.
- ❌ **시간 초과 무시** — 4분 안에 안 끝나면 진짜 감점. 끝낼 타이밍 미리 정하기.
- ✅ **리허설 최소 1회.**
- ✅ **DB를 깨끗한 시드 상태로** — 발표 직전 1번.
- ✅ **발표자 2명 권장** — 1명이 agent UI, 1명이 customer UI 동시에 만질 수 있게.
- ✅ **마지막 30분은 polish만, 새 시도 없음.**

---

## 7. 자주 만나는 함정 / Common pitfalls

| 함정 | 증상 | 해결 |
|---|---|---|
| LLM이 너무 느림 | 첫 토큰 3초+ | prompt 짧게, max_tokens 작게, stream 켜기 |
| STT가 한국어 인식 못함 | transcript가 영어 또는 빈값 | Transcribe `LanguageCode=ko-KR` 명시 |
| iPhone UI가 안 울림 | 데모에서 "전화 옴" 표현 약함 | `afplay`로 .m4a 링톤 재생 (Mac) |
| 노드 그래프가 너무 작아 안 보임 | 데모 projector에서 글씨 안 보임 | 폰트 키우기 (tw-2xl), 줌 디폴트 1.5배 |
| 통화 중 backend 죽음 | 30분 데모 중 1번 | supervisor로 자동 재시작, 또는 발표 전 재시작 |
| AI 인계 요약의 LLM 내용이 별로 | "통화가 종료되었습니다" 같은 일반론 | system prompt에 "구체적: 고객이 X에 대해 Y라고 말함" 강조 |
| 통합 충돌 | BE/FE schema 다름 | 발표 직전 1시간은 통합 fix만 |
